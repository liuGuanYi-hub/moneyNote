🚀 记账助手 Pro (MoneyNote) 创作历程：从零到一的全栈与 AI 蜕变
“记账不应该是一件枯燥的苦差事，而应该是一场充满成就感的财务养成游戏。”

这是我在开发《记账助手 Pro》时立下的初心。回望整个项目的开发历程，这不仅仅是写下一千多行代码，更是我从“传统 CRUD”走向“AI Agent（智能体）”与“高级数据可视化”的一段全栈进阶之旅。

💡 阶段一：破土而出 —— 架构与核心搭建
项目的起点在于构建一个稳定、可靠的地基。我选择了最扎实的主流技术栈：

后端：Java + Spring Boot + Spring Security (JWT)

前端：HTML5 + CSS3 + 原生 JavaScript + Apache ECharts

数据库：MySQL (Aiven 云端数据库)

AI 引擎：阿里云百炼大模型 API (通义千问 Qwen)

在这一阶段，我打通了任督二脉：

API 闭环：设计了清晰的 RESTful API，实现了用户鉴权、分类管理和账单 CRUD。

移动优先 (Mobile First)：前端完全采用响应式设计，完美适配手机屏幕，采用类 App 的 Tab 栏导航，为后续体验打下基础。

🛠️ 阶段二：打磨细节 —— 从“能用”到“好用”
当基础功能跑通后，我开始站在用户的角度思考：商业软件好在哪？ 于是，我为应用注入了商业级的细节：

♻️ 回收站机制（软删除）：手滑删错账单是常态。我不仅做了回收站页面，还在后端实现了“软删除与恢复”的完整闭环，给用户后悔的权利。

📁 纯前端 CSV 导出：为了保护用户隐私，无需后端介入，通过纯 JS 的 Blob 对象，直接在浏览器端生成兼容 Excel 的账单表格。

🤖 智能预算监控：系统会根据当月剩余天数，动态计算出**“日均建议消费额”**，并结合颜色（绿、黄、红）给出财务体检报告。

🎮 阶段三：视觉革命 —— 数据大屏与游戏化
这是项目颜值和体验产生质变的阶段！我用顶级的前端库为项目注入了高级感。

1. 📊 专业级数据可视化 (ECharts)
   我摒弃了传统的静态图表，引入了百度开源的 Apache ECharts：

悬浮交互：实现了动态中空环形饼图与平滑柱状图，手指点击即可精准查看每一分钱的去向。

动画自适应：图表加载时带有炫酷的数据生长动画，且能完美自适应手机屏幕的旋转。

2. 🏆 勋章墙系统
   我设计了 7 个维度的进阶勋章，配合动态进度条，让记账变成打怪升级：

“深夜食堂”（22点后记餐饮）、“土豪勋章”（单笔超1000）、“精打细算”（结余率超30%）。

3. ✨ 大厂级微动效 (Micro-interactions)
   💰 撒金币特效：引入 canvas-confetti 引擎。记账或解锁成就的一瞬间，屏幕喷射黄金纸屑。

📈 数字滚动算法：利用 requestAnimationFrame 编写缓动函数，首页结余金额飞速滚动。

🧠 阶段四：AI 觉醒 —— 步入 Agent 时代 (高光时刻)
手动选择分类、输入金额依然是一种负担。为了彻底解放双手，我引入了大语言模型 (LLM)，将记账本升级为了 AI Agent。

NLP 语义理解：对接阿里云通义千问 API，运用 Prompt Engineering (提示词工程) 规范 AI 的输出。

极简体验：用户只需打出一句口语化的大白话，例如：“昨天买了两杯瑞幸咖啡，一共 38 块”。

自动化入库：后端 AI 模块会自动提取出 金额：38、分类：餐饮、备注：瑞幸咖啡，并智能匹配本地数据库进行自动记账。

这一功能让系统的易用性产生了质的飞跃。

☁️ 阶段五：拥抱云端 —— 部署与上线
本地跑通不是终点，能让朋友用上才是。 我将代码托管至 GitHub，利用 Render 平台搭建了免费的自动化构建流水线，并连接了 Aiven 云端数据库。 看着绿色的 Pushed successfully 和 Render 上线成功的日志，属于我自己的第一个全栈 AI 云端产品正式诞生了。

📝 总结与展望
技术收获清单：

✅ 深刻理解了前后端分离架构与 JWT 鉴权流程。

✅ 掌握了原生 JS 操作 DOM 与处理异步 Fetch 请求的高级技巧。

✅ 掌握了 ECharts 的高级配置与 Canvas 动画原理。

✅ 掌握了对接大模型 API 的方法，实现了从传统 CRUD 到 AI 智能应用的技术跨越。

下一步计划： 由于项目的基础架构已经非常完善，未来我将向更高级的移动端体验进发：

PWA 离线应用化：添加 manifest.json，让网页可以像原生 App 一样免安装添加到手机桌面。

纯语音记账：引入 HTML5 Web Speech API，实现真正的“动口不动手”，语音直接投喂给 AI。
